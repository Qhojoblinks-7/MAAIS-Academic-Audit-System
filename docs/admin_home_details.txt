================================================================================
          DETAILED ARCHITECTURAL GUIDE: THE ADMIN HOME PAGE
================================================================================

This document describes the design architecture, visual components, interactive
controls, and system tracking features of the central Admin Home Page 
(AdminHome.jsx) in the Administrative panel.

--------------------------------------------------------------------------------
1. GLOBAL OBJECTIVE
--------------------------------------------------------------------------------
The Admin Home Page serves as the core high-fidelity command deck for school 
administrators. It combines beautiful Swiss-modern aesthetics with live 
analytical charts, systemic tracking telemetry, and real-time operations to 
keep the entire institution synchronized.

--------------------------------------------------------------------------------
2. CURRENT STATE HEADER (LIVE NODE TRACKER & CONFIGURATION)
--------------------------------------------------------------------------------
The topmost card provides the administrator with an instantaneous view of active
academic sessions and system health, with interactive controls to reconfigure it:
  - Live UTC Clock: Built upon a React hook interval updating every second to 
    maintain high-contrast, tabular-nums precision.
  - Active Session & Target Level Switcher:
    - Instead of static read-only indicators, the entire Academic Year, Term, and 
      Target Student Level (e.g., "SHS 3") block serves as an interactive switcher.
    - Clicking the button opens a high-fidelity "Configure Academic Node" modal.
    - Inside this modal, administrators can dynamically shift the active tracking
      target across:
      - Student Levels: SHS 1, SHS 2, SHS 3, SHS 4
      - Academic Years: 2024/2025, 2025/2026, 2026/2027
      - Academic Terms: Term 1, Term 2, Term 3
    - Modifying these parameters instantly updates the system-wide contextual tracking
      state across the home view dashboard.
    - Features a glowing, pulsing green "System Live" indicator representing successful 
      real-time database sync.

--------------------------------------------------------------------------------
3. VITAL SIGNS (THE PULSE METRICS)
--------------------------------------------------------------------------------
Below the header is a row of four high-fidelity metric panels tracking 
institutional health metrics:
  - Student Census: Shows total student enrollment, categorized by Boarders vs. 
    Day student counts.
  - Faculty Engagement: Displays active tutors and immediately flags offline or 
    unassigned faculty members.
  - Grading Progress: Measures grading performance across the school using an 
    interactive progress velocity gauge that changes color based on progress tier 
    (Red: <40%, Amber: 40-75%, Green: >75%).
  - Flagged Activities: Real-time telemetry monitoring integrity alerts or database 
    anomalies.
  - Sparklines: Each card incorporates a custom, animated vector path SVG mapping 
    historical performance trends dynamically.

--------------------------------------------------------------------------------
4. CENTRAL VISUALIZATION: DEPARTMENTAL PERFORMANCE HEATMAP
--------------------------------------------------------------------------------
To analyze performance and educational efficiency, the main body displays a 
large, custom-styled charting component:
  - Grid Framework: Utilizes Recharts with responsive container sizing to map 
    average grades across core divisions: Science, Business, General Arts, 
    Home Economics, and Visual Arts.
  - Custom Accents: Columns are plotted with custom-rounded, translucent bars 
    shaded in distinct, randomized departmental color themes.
  - Micro-Interactions: An interactive hover tooltip displays the specific 
    Department Name, raw percentage score, and academic health quotient on mouse-over.
  - Dynamic Highlights: High-performing departments (scores exceeding 80%) 
    automatically trigger upward trend indicators (`ArrowUpRight`).

--------------------------------------------------------------------------------
5. OPERATIONAL SIDEBARS (RIGHT CONTROL RAIL)
--------------------------------------------------------------------------------
The right column groups administrative queue tasks into digestible modules:
  - Pending Approvals Queue: Monitors high-impact faculty submissions (e.g. 
    grade updates, bulk deletions). Hovering over cards reveals instant action buttons 
    to "Grant" (with a ThumbsUp green overlay) or "Abort" (with a ThumbsDown red overlay).
  - Network Support Queue: Tracks system syncing states and pending IT support tickets 
    (e.g., tablet syncing, database loops) with priority visual rings.
  - Strategic Memo Notepad: An interactive notepad bound to a local React state 
    allowing admins to log qualitative notes, checklists, and reminders during 
    their sessions.

--------------------------------------------------------------------------------
6. REGISTRY ACTIVITY FEED (AUDIT TRAIL)
--------------------------------------------------------------------------------
Located at the bottom of the main dashboard, this dark Cosmic-styled section 
renders a continuous feed of system-wide logs:
  - Activity Logs: Tracks operations like grading finalizations, bulk SMS 
    dispatches, student allocations, and server checks.
  - Category Mapping: Logs are categorized by type (academic, comm, system, 
    security, support) with precise timestamps.
  - Deep Navigation: Provides a quick link to navigate directly to the Extended 
    Audit Logs module for surgical investigations.

--------------------------------------------------------------------------------
7. THE FLOATING QUICK ACTION HUB (OMNI-CHANNEL CONTROLS)
--------------------------------------------------------------------------------
A floating action button (FAB) at the bottom-right corner reveals a custom menu of 
high-frequency actions:
  - Register Node: A multi-profile wizard allowing admins to instantly jump to the 
    Students, Faculty, or Parents registries to provision new user credentials.
  - Broadcast Pulse: An overlay modal that enables administrators to compose 
    system communications and dispatch them instantly across multiple channels 
    (In-App Push, Bulk SMS, or Academic Email).
  - Emergency Freeze: The ultimate systemic safety switch. In case of security or 
    audit concerns, this lock model freezes all database write actions across all 
    faculty channels. When active, the main FAB changes into a pulsing red lock 
    icon, indicating system-wide read-only status.
